USER MANUAL LINE Vehicle Verification

คู่มือการใช้งานระบบตรวจสอบทะเบียนรถผ่าน LINE Bot

เอกสารฉบับนี้อัปเดตตามโค้ดและ feature ล่าสุดของระบบ รวมทั้งการค้นหาทะเบียน, OCR, คำสั่ง admin, health checks, alerts และโครงสร้างชีตที่ใช้งานจริงในปัจจุบัน

ผู้ใช้งานทั่วไป

ใช้ /myid และ /help ได้ทันที แม้ยังไม่ได้รับสิทธิ์ในระบบ

เจ้าหน้าที่และลูกบ้านที่ได้รับสิทธิ์

ค้นหาทะเบียนรถ, บ้านเลขที่ และส่งรูปเพื่อ OCR ได้ในกลุ่มที่อนุญาต

Admin

จัดการสิทธิ์, ตรวจสุขภาพระบบ, ดู system logs และทดสอบ alert pipeline ได้จาก LINE

เริ่มต้นใช้งาน ค้นหาข้อมูลรถ คำสั่ง Admin สิทธิ์ผู้ใช้งาน โครงสร้างข้อมูล สุขภาพระบบและปัญหาที่พบบ่อย

วิธีเริ่มใช้งานระบบ

01
เพิ่ม Bot หรือเปิดแชตกับ Bot

ระบบนี้ทำงานบน LINE Bot ผู้ใช้ทุกคนสามารถเริ่มด้วยการส่ง /myid เพื่อดู User ID ของตัวเองได้ก่อน

02
ส่ง /help เพื่อดูคำสั่งที่ใช้ได้

ถ้าเป็น admin ระบบจะโชว์คำสั่งสำหรับจัดการข้อมูลและ monitoring เพิ่มเติมให้อัตโนมัติ

03
ถ้ายังไม่มีสิทธิ์ ให้ส่ง User ID ให้ผู้ดูแล

Admin จะนำ User ID ไปเพิ่มในชีต Staff พร้อมกำหนด role และสถานะการใช้งาน

/myid
/help
หมายเหตุ: ผู้ใช้ที่ยังไม่ได้ลงทะเบียนจะยังใช้ค้นหาข้อมูลรถไม่ได้ แต่ยังดู /myid และ /help ได้

วิธีค้นหาข้อมูลรถ

ระบบรองรับทั้งการค้นหาจากข้อความและจากรูปภาพทะเบียนรถ โดยผลลัพธ์จะอิงข้อมูลในชีต Vehicles

ตัวอย่างทะเบียนและบ้านเลขที่ในคู่มือนี้เป็นข้อมูลสมมติสำหรับอธิบายวิธีใช้งานเท่านั้น

ค้นหาด้วยข้อความ

  • พิมพ์ทะเบียนตรง ๆ เช่น ทด1234
  • พิมพ์ทะเบียนแบบมีขีดหรือช่องว่าง เช่น 80-0001, 3 ทฮ 7007
  • พิมพ์บ้านเลขที่ เช่น 10/23
ทด1234
80-0001
10/23

ค้นหาด้วยรูปภาพ

  • ส่งรูปป้ายทะเบียนในแชตกลุ่มที่อนุญาต
  • ระบบจะดึงภาพจาก LINE Content API แล้วส่งต่อให้ Gemini ช่วยอ่านทะเบียน
  • ถ้าพบเลขใกล้เคียง ระบบจะแสดงรายการที่อาจเป็นและสรุปผลตรวจให้

ผลลัพธ์ที่อาจพบ

พบข้อมูลและรถ active ไม่พบตรงตัว แต่มีรายการใกล้เคียง ไม่พบข้อมูล หรือ OCR อ่านไม่ชัด
ข้อความตอบกลับผู้ใช้งานยังคงเป็นภาษาไทยตามการใช้งานจริงของหน้างาน

คำสั่ง Admin

คำสั่งด้านล่างใช้ได้เฉพาะผู้ใช้ที่มี role เป็น admin เท่านั้น

คำสั่ง หน้าที่ ตัวอย่าง
/add <userId> <name> <role>เพิ่มผู้ใช้ในชีต Staff/add U12345 Somchai staff
/remove <userId>ลบผู้ใช้ออกจากระบบ/remove U12345
/setstatus <userId> <active|inactive>เปลี่ยนสถานะการใช้งาน/setstatus U12345 active
/setrole <userId> <admin|staff>เปลี่ยน role ของผู้ใช้/setrole U12345 admin
/listดูรายชื่อ staff และ admin/list
/status <userId>ดูข้อมูลผู้ใช้รายคน/status U12345
/whoisดูข้อมูลผู้ส่งคำสั่งปัจจุบัน/whois
/visitorsดูผู้ใช้งานล่าสุดจากชีต Visitors/visitors
/log <count>ดู search log ล่าสุด/log 10
/syslog <count>ดู SystemLog ล่าสุด/syslog 10
/healthตรวจสุขภาพระบบแบบเบา ไม่ยิง LINE/Gemini live check/health
/health fullตรวจสุขภาพระบบแบบเต็ม รวม live check ของ LINE และ Gemini/health full
/testalertทดสอบ admin alert pipeline และเขียน SystemLog ทันที/testalert
/clearcacheล้าง cache ที่เกี่ยวกับข้อมูลระบบ/clearcache
/versionดู version และ SHA ปัจจุบัน/version

คำสั่งสำหรับ monitoring

  • /health ใช้ตรวจ config, spreadsheet, cache และ drive แบบเร็ว
  • /health full ใช้ตอน debug เมื่ออยากรู้ว่า LINE API และ Gemini API ใช้งานจริงได้หรือไม่
  • /syslog 10 ใช้ดู system events ล่าสุดโดยไม่ต้องเปิด Google Sheets
  • /testalert ใช้ยืนยันว่า LINE admin alert และ SystemLog ยังทำงานอยู่

ประเภทผู้ใช้งานและสิทธิ์

ประเภท สิ่งที่ทำได้ ข้อจำกัด
ยังไม่ลงทะเบียน /myid, /help ยังค้นหาข้อมูลและใช้ OCR ไม่ได้
staff ค้นหาทะเบียน, ค้นหาบ้านเลขที่, ส่งรูป OCR ต้องใช้งานในกลุ่มที่อยู่ใน ALLOWED_GROUP_IDS เว้นแต่จะเป็น admin
admin ใช้ทุกคำสั่ง รวมทั้ง monitoring และ maintenance support ไม่มีข้อจำกัดเรื่อง allowlist group สำหรับงานค้นหา

โครงสร้าง Spreadsheet

ชีต หน้าที่ หมายเหตุ
Staffเก็บชื่อ, UID, status, roleเป็น master data สำหรับสิทธิ์ผู้ใช้
Vehiclesเก็บทะเบียนรถ, บ้านเลขที่, เจ้าของ, สถานะรถใช้ทั้ง search และ OCR matching
Visitorsเก็บผู้ใช้ที่เคยใช้งานระบบมี row map cache เพื่อให้เขียนเร็วขึ้น
Logเก็บ search logsระบบ buffer ก่อนเขียนลงชีต
SystemLogเก็บ events ระดับระบบ เช่น health summary, alert, failuresถ้ายังไม่มี ระบบสามารถสร้างให้อัตโนมัติพร้อม header

สุขภาพระบบและปัญหาที่พบบ่อย

ระบบ log และ alert ทำงานอย่างไร

  • Log ใช้เก็บประวัติการค้นหาและจะ buffer ก่อน flush ลงชีต
  • SystemLog ใช้เก็บ operational events
  • ระดับ ALERT และ ERROR จะถูก flush ลง SystemLog ทันที
  • /testalert ใช้ทดสอบทั้ง LINE alert และ SystemLog โดยไม่ต้องทำระบบจริงให้ fail

กรณีที่พบบ่อย

อาการ สิ่งที่ควรเช็ก วิธีแก้เบื้องต้น
Bot ไม่ตอบWebhook, deployment, Script Propertiesเริ่มจาก /health และเช็ก deployment ล่าสุด
OCR อ่านไม่ออกความชัดของภาพ, Gemini API, rate limitลองถ่ายใหม่ หรือใช้ /health full เพื่อตรวจ Gemini
ไม่เห็น system event ในชีตSystemLog, ระดับ log, bufferใช้ /syslog 10 หรือ /testalert เพื่อยืนยัน pipeline
ข้อมูลในชีตแก้แล้วแต่ผลยังไม่อัปเดตcache ของ Staff, Vehicles, Visitorsรอ cache refresh หรือให้ admin ใช้ /clearcache
maintenance ทำงานไม่ครบDrive access, backup folder, retention jobsดู LINE admin alert และ SystemLog event maintenance_partial_failure
แนะนำให้ใช้ชุดคำสั่ง /health, /syslog 10 และ /testalert เป็น baseline เวลาตรวจ staging หรือหลัง deploy
อัปเดตตาม codebase ปัจจุบันของโปรเจกต์ Vehicle Verification System บน Google Apps Script และ LINE Bot
