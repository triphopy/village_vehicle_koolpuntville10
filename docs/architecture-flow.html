System Overview

Webhook entry

doPost(e) รับ event, parse payload และสร้าง requestId สำหรับการ trace

Handlers

textHandler, imageHandler และ followHandler แยก flow ตามชนิดของ event

Ops commands

/health, /syslog, /testalert ช่วยดูแลระบบจากใน LINE โดยตรง

Logging

Log และ SystemLog มี buffering และระดับความสำคัญต่างกัน

flowchart TD
    U["LINE User"] --> LINE["LINE Messaging API"]
    LINE --> WEBHOOK["GAS Web App doPost(e)"]
    WEBHOOK --> PARSE["parse request + requestId"]
    PARSE --> ROUTER["routeLineEvent"]

    ROUTER --> FOLLOW["followHandler"]
    ROUTER --> TEXT["textHandler"]
    ROUTER --> IMAGE["imageHandler"]

    TEXT --> STAFF["staffService"]
    TEXT --> SEARCH["vehicleSearchService"]
    TEXT --> ADMIN["adminCommandRouter"]
    TEXT --> VISITOR["visitorService"]
    TEXT --> LOG["logService"]

    IMAGE --> OCR["ocrService"]
    IMAGE --> SEARCH
    IMAGE --> VISITOR
    IMAGE --> LOG

    ADMIN --> HEALTH["healthCommand"]
    ADMIN --> SYSLOG["syslogCommand"]
    ADMIN --> TESTALERT["testAlertCommand"]

    STAFF --> SHEETS["Google Sheets"]
    SEARCH --> SHEETS
    VISITOR --> SHEETS
    LOG --> SHEETS
    OCR --> GEMINI["Gemini API"]
    OCR --> LINECONTENT["LINE Content API"]

    G -- "no" --> H["Reply temporary unavailable"]

    N -- "yes" --> O["adminCommandRouter"]

    P --> Q["Reply to LINE"]

    S --> T["Reply to LINE"]

Maintenance and Deployment

Maintenance

dailyMaintenance() flush logs, สำรองข้อมูล, cleanup และส่ง alert ถ้ามี partial failure

Retention

dailyCleanup() ล้างข้อมูลเก่าจาก Log, Visitors และ SystemLog

Deployment

push ไป main เพื่อ production และ feature/** เพื่อ staging ผ่าน GitHub Actions + clasp

flowchart TD
    A["Developer pushes to main or feature/**"] --> B["GitHub Actions deploy.yml"]
    B --> C["Install clasp"]
    C --> D["Inject src/versionInfo.js"]
    D --> E["clasp push --force"]
    E --> F["clasp deploy"]
    F --> G{"branch = main?"}
    G -- "yes" --> H["BLUE / Production GAS"]
    G -- "no" --> I["GREEN / Staging GAS"]
